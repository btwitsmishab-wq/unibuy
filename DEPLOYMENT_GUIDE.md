# ☁️ UniBuy Enterprise Cloud Deployment Guide

This guide details the production deployment architecture for the UniBuy application using industry-standard cloud providers (Amazon Web Services or Microsoft Azure). This architecture ensures scalability, reliability, and meets academic/enterprise evaluation criteria.

## Option A: Amazon Web Services (AWS) Architecture 🏆

AWS is the industry standard for cloud computing. This setup uses fully managed AWS services.

### 1. Database: Amazon RDS (Relational Database Service)
1. Log in to the [AWS Management Console](https://console.aws.amazon.com/).
2. Navigate to **RDS** and click **Create database**.
3. Select **PostgreSQL** and choose the **Free tier** template.
4. Set the DB instance identifier (e.g., `unibuy-db`), master username, and master password.
5. Under Connectivity, ensure **Public access** is set to **Yes** (or configure a VPC if preferred).
6. Click **Create database**. Save the endpoint URL, username, and password. Your connection string will look like: 
   `postgresql://username:password@endpoint-url:5432/postgres`

### 2. Backend API: AWS Elastic Beanstalk / App Runner
1. Navigate to **AWS App Runner** (easiest for Node.js) or **Elastic Beanstalk**.
2. Click **Create an App Runner service**.
3. Point it to your GitHub repository containing the Node.js backend.
4. Set the build command: `npm install` and start command: `npm start`.
5. Under **Environment variables**, add:
   - `DATABASE_URL`: Your Amazon RDS connection string.
   - `FIREBASE_SERVICE_ACCOUNT`: Your Firebase admin credentials (if used).
6. Deploy the service and copy the provided AWS URL (e.g., `https://xyz.awsapprunner.com`).

### 3. Frontend: AWS Amplify
1. Build your Flutter web app locally:
   ```powershell
   flutter build web
   ```
2. Navigate to **AWS Amplify** in the AWS Console.
3. Choose **Host web app** and select **Deploy without Git provider** (or connect your repo).
4. Upload the `build/web` folder generated by Flutter.
5. Amplify will deploy your app and provide a continuous deployment URL (e.g., `https://main.xyz.amplifyapp.com`).

## Option B: Microsoft Azure Architecture 🏢

Azure is a highly respected enterprise cloud platform.

### 1. Database: Azure Database for PostgreSQL
1. Log in to the [Azure Portal](https://portal.azure.com/).
2. Search for **Azure Database for PostgreSQL** and create a new Flexible Server.
3. Select your subscription, resource group, and provide a server name.
4. Choose the **Development** or **Free** tier workload.
5. Create an admin username and password.
6. Under Networking, allow public access from any Azure service and your IP.
7. Once provisioned, copy the connection string from the resource dashboard.

### 2. Backend API: Azure App Service
1. In the Azure Portal, create a new **Web App**.
2. Set the Publish mode to **Code**, Runtime stack to **Node.js (LTS)**, and Operating System to **Linux**.
3. Choose the **Free (F1)** pricing plan.
4. In the **Deployment Center**, link your GitHub repository.
5. In **Configuration** -> **Application settings**, add your environment variables (`DATABASE_URL`, etc.).
6. Save and let Azure build and deploy your Node.js backend. Save the `.azurewebsites.net` URL.

### 3. Frontend: Azure Static Web Apps
1. Build your Flutter web app: `flutter build web`.
2. In the Azure Portal, create a new **Static Web App**.
3. Connect your GitHub repository.
4. For Build Details, choose **Custom**. Set the App location to `/` and Output location to `build/web`.
5. Azure will generate a GitHub Actions workflow to automatically build and deploy your Flutter app.
6. Share the generated Azure URL with your evaluator!

---

## Final Step: Connect Frontend to Backend

Regardless of whether you chose AWS or Azure, you must update your Flutter app to point to the new cloud backend.

1. Open `lib/config.dart`.
2. Ensure `isProduction = true;`.
3. Set `productionBackendUrl` to your new AWS App Runner or Azure App Service URL:
   ```dart
   static const String productionBackendUrl = "https://your-new-cloud-url.amazonaws.com"; // or azurewebsites.net
   ```
4. Rebuild the frontend (`flutter build web`) and deploy it to AWS Amplify or Azure Static Web Apps!
